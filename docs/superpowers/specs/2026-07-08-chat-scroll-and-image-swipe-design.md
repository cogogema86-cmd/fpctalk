# 채팅 초기 스크롤 수정 + 이미지 뷰어 스와이프 갤러리

날짜: 2026-07-08
상태: 사용자 승인 완료

## 배경

1. 채팅방 진입 시 "사용자가 마지막으로 읽은 위치"부터 보여야 하는데, 계속 중간 부분부터 표시됨.
2. 모바일에서 채팅 사진을 열었을 때 좌우 스와이프로 다른 사진을 넘겨볼 수 없음.

## 1. 초기 스크롤 버그 수정

### 원인

`lib/chat.ts`의 `getChatMessages`가 `orderBy: { createdAt: "asc" }, take: 200`으로 조회
→ 메시지가 200개를 초과하는 방에서는 **가장 오래된 200개**만 로드됨.
안읽음 구분선 대상(최근 메시지)이 로드 범위 밖이라 구분선을 못 찾고,
"맨 아래로 스크롤"도 사실상 채팅 이력의 중간 지점이었음.

### 수정

- `orderBy: { createdAt: "desc" }, take: 200` 조회 후 `reverse()`로 시간순 정렬해 반환.
- 기존 클라이언트 로직(`_chat-room.tsx`)은 그대로 동작:
  - 안 읽은 메시지 있음 → "여기서부터 안 읽음" 구분선으로 스크롤
  - 없음 → 맨 아래(최신 메시지)

## 2. 이미지 뷰어 스와이프 (채팅방 전체 사진 — 카카오톡 방식)

### 구조 변경

- **갤러리 목록**: `ChatRoom`이 로드된 메시지들에서 만료되지 않은 이미지 첨부 전체를
  시간순 평면 목록(`{messageId, attIdx, name}`)으로 계산.
- **뷰어 상태 리프트**: 말풍선(`MessageBubble`)별 개별 뷰어 → `ChatRoom` 레벨 단일 뷰어.
  말풍선은 `onOpenImage(messageId, attIdx)` 콜백만 호출.

### ImageViewer 확장

- props: 사진 목록(items) + 시작 인덱스(initialIndex), 내부에서 현재 인덱스 관리
- 터치 스와이프: 가로 이동 50px 이상이면 이전/다음 (세로 스크롤 제스처와 구분)
- 데스크톱: 좌우 화살표 버튼 + 키보드 ←/→
- 상단에 `3 / 12` 위치 카운터
- 공유/저장은 현재 사진 기준 (기존 blob 미리 준비 로직 재사용 — URL 변경 시 재실행)
- 인접 사진 미리 로드

### 제한사항

- 로드된 최근 200개 메시지 범위의 사진만 탐색 가능 (현재 페이지네이션 구조상 한계).
- 외부 라이브러리 미사용 — 기존 코드 스타일대로 직접 구현.
